# What I Will Need From You To Go Live

Use this later when you are ready to publish WillChecklist.com.

## 1. Domain and hosting
- Domain registrar login or DNS access for `willchecklist.com`.
- Hosting/cPanel/static host login, or the name of the host you want to use.
- Confirmation of whether the site should be served from `https://noahbsark.github.io/willchecklist/` and whether `www.willchecklist.com` should redirect to the root domain.

## 2. Public mode decision
- Keep public-test mode on, or turn on newsletter/lead/affiliate/ad blocks.
- Whether to upload only the public `/site` folder.
- Whether any Markdown owner files should be excluded from the public upload.

## 3. Email/newsletter
- Email provider choice, such as Mailchimp, Beehiiv, ConvertKit, Brevo, or another provider.
- Signup form endpoint or embed code.
- Sender name and sender email address.
- Whether to send the PDF packet immediately after signup.

## 4. Forms/contact
- Contact form endpoint or form service.
- Email address that should receive messages.
- Whether the planning-help page should stay hidden until a real intake process exists.

## 5. Analytics/search
- Analytics provider preference.
- Analytics ID or script, if you want analytics.
- Google Search Console verification method when ready.
- Sitemap submission after upload: `https://noahbsark.github.io/willchecklist/sitemap.xml`.

## 6. Trust and legal review
- Whether an estate-planning attorney will review the core tools and PDFs.
- Name, credentials, review date, and exact scope only if a real review happens.
- Any state-specific content should stay avoided unless professionally reviewed and maintained.

## 7. Monetization
- Which affiliate categories you want to pursue, if any.
- Approved affiliate links only after joining programs.
- Ad network approval details before adding ads.
- Whether to keep monetization hidden during initial public testing.

## 8. Brand/contact details
- Public contact email address.
- Business/entity name, if different from Will Checklist.
- Mailing address only if you want/need to publish one.
- Preferred privacy-policy contact language.

## 9. Final launch check
- Test the homepage on phone and desktop.
- Test Start → Continue → Progress → Summary.
- Confirm no placeholder forms are visible unless connected.
- Confirm disclosures remain visible if monetization is activated.
- Confirm no private builder notes are uploaded.
