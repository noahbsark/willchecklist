# Public Test Readiness

Use this checklist before handing the site to a real user.

## Ready for test

- The homepage has one obvious first action: Start Step 1.
- The Start navigation goes directly to the setup wizard.
- Continue and Progress use browser-local saved completion states.
- Core pages end with one main next-step button.
- Optional copy, print, download, and reset actions are tucked into utility drawers where possible.
- Inactive ads, affiliate links, newsletter forms, and lead forms should remain hidden or paused.

## What to watch during a test

- Does the user know what to click first?
- Does the user understand they can skip questions?
- Does the user avoid entering sensitive information?
- Does the user understand that the site does not create a legal document?
- After finishing a step, does the user know what to click next?

## Do not test with real secrets

Ask testers to use fake or generic example notes. They should not enter passwords, Social Security numbers, account numbers, access codes, private keys, seed phrases, or confidential legal facts.
