# Owner Guide for WillChecklist.com

This guide explains the simplest way to keep the site clean and usable.

## Public upload
Upload only the contents of the public site folder.

## Primary user path
1. Home page
2. Setup
3. Continue / Dashboard
4. Checklist and document locator
5. Summary Builder
6. Quick-start PDF or professional meeting prep

## Keep the GUI simple
- Keep the top navigation short: Home, Start, Continue, Dashboard, Help, Print guide.
- Avoid adding more top-level links unless one replaces an existing link.
- Put extra articles in the footer or sitemap, not the main nav.
- Every tool page should have one obvious next step.

## Safety rule
Never ask visitors to enter passwords, Social Security numbers, full account numbers, safe codes, private keys, seed phrases, or confidential legal facts.

## Before launch
- Connect or hide placeholder email and lead forms.
- Replace affiliate placeholders only with real links and clear disclosures.
- Get attorney review before claiming legal review.


## V13 public-test update

This version pauses public newsletter, lead, affiliate, and display-ad placeholders so real users do not hit dead submissions or misleading monetization. Use the site as a working planning tool first. See `PUBLIC_TEST_READINESS.md`, `USABILITY_TEST_SCRIPT.md`, and `OWNER_LAUNCH_INPUTS_DRAFT.md`.


## V14 simplicity rules

Keep the main navigation limited to Start, Continue, Dashboard, Help, and Print Guide. Do not add every worksheet to the main nav. The site should feel like one guided product, not a library of pages.

Each core tool page should keep the same pattern: what this step does, safe to enter, do not enter, the task, mark done, next step.


## V15 simplicity rule

Do not add more items to the main navigation. Keep the public path to Start, Continue, Progress, Help, and Print Guide. Put optional tools in the footer, sitemap, or inside the Progress page details section.
